# Fabric 1.20.2 Startup Fix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 Minecraft 1.20.2 的 Fabric 专用服使用正确版本的 Cloth Config 启动，不再加载面向 1.20.4 的 Fabric API Registry Sync mixin。

**Architecture:** `fabric/build.gradle` 已从版本属性读取 `cloth_config_version` 并将其用于编译与本地开发运行时。只将 1.20.2 的属性由错误的 1.20.4 Cloth Config 工件改为明确支持 1.20.2 的 `12.0.137`，不引入排除规则或改变运行时逻辑。

**Tech Stack:** Gradle 8.14.5、Architectury Loom、Fabric Loader 0.19.3、Fabric API 0.91.6+1.20.2、Cloth Config。

## Global Constraints

- 仅改 `versionProperties/1.20.2.properties` 的 `cloth_config_version`。
- `fabric_api_version=0.91.6+1.20.2` 必须保持不变。
- `modmenu_version=8.0.1` 必须保持不变。
- 禁止修改 Hassium 网络、存储、Mixin、配置业务逻辑或其他 Minecraft 版本属性。
- 验证目标是 1.20.2 Fabric 专用服真实启动至 `Done (...)!`。

### Task 1: 固定 1.20.2 的兼容 Cloth Config 版本

**Files:**
- Modify: `versionProperties/1.20.2.properties:29`
- Test: 1.20.2 Fabric Gradle 编译与专用服启动

**Interfaces:**
- Consumes: `fabric/build.gradle` 中 `gradle.ext.cloth_config_version` 对 `modCompileOnly` 与 `modLocalRuntime` 的既有解析。
- Produces: 解析到 `me.shedaniel.cloth:cloth-config-fabric:12.0.137` 的 1.20.2 Fabric 开发运行时。

- [ ] **Step 1: 写入兼容工件版本**

将 `versionProperties/1.20.2.properties` 中：

```properties
cloth_config_version=13.0.138
```

替换为：

```properties
cloth_config_version=12.0.137
```

不改同一文件的 `fabric_api_version=0.91.6+1.20.2`、`modmenu_version=8.0.1` 或任何其他版本属性。

- [ ] **Step 2: 编译目标变体**

Run:

```powershell
./gradlew --no-daemon fabric:compileJava "-Pmc_ver=1.20.2"
```

Expected: `BUILD SUCCESSFUL`，且 `:fabric:compileJava` 成功完成。

- [ ] **Step 3: 真实启动 Fabric 专用服**

Run:

```powershell
./gradlew --no-daemon fabric:runServer "-Pmc_ver=1.20.2"
```

在服务端日志出现 `Done (...)!` 后停止该测试进程。

Expected: 日志不包含 `SimpleRegistryMixin`、`InvalidMixinException` 或 `field_26683`；服务端完成启动。

- [ ] **Step 4: 记录验证结果**

检查 `fabric/run/server/logs/latest.log`：确认已加载 Fabric API `0.91.6+1.20.2`，并已出现服务端完成启动标记。若开发运行目录保留的是旧工件，先使用新的 Gradle 依赖解析结果重新启动；不删除用户配置或世界数据。

- [ ] **Step 5: 提交**

```bash
git add versionProperties/1.20.2.properties
git commit -m "fix: use 1.20.2 Cloth Config on Fabric"
```

提交仅在用户要求创建提交时执行。
